# Superclasses 요약

메서드 상속 기능을 구현한다. 먼저 상속 구문의 컴파일을 구현한다. 
상속 구문을 발견하면 슈퍼 클래스를 찾아서 런타임에 사용할 수 있도록 스택에 클래스를 올리는 바이트코드를 에밋하고 OP_INHERIT 을 에밋한다.
런타임에 클래스 선언을 인터프리팅 할때 OP_INHERIT 이 있으면 먼저 슈퍼클래스의 메서드 테이블을 서브클래스에 복사한다.
이후 메서드 선언이 인터프리팅 되면서 메서드가 메서드 테이블에 추가되기 때문에 자연스럽게 오버라이딩된다.
JLox 에서는 서브클래스가 슈퍼클래스에 대한 참조를 가지고 있고, 참조를 거슬러 올라가서 메서드를 찾았지만,
CLox 에서는 메서드 테이블 자체가 복사됨으로 훨씬 간단하고 빠르다. 이를 copy-down inheritance 라고 부른다.

슈퍼 클래스의 메서드를 호출 할 수 있는 기능을 구현 하기 위해서 먼저 슈퍼 클래스를 저장하는 기능을 구현한다.
슈퍼 클래스에 대한 참조는 정적으로 렉시컬 스코프를 가진다.
this 와 비슷하게 컴파일타임에 스코프를 열고 super 를 지역변수로 등록한다. upvalue 시스템을 그대로 사용한다.
super 토큰을 컴파일 할때, super 는 단독으로 사용될 수 없고 수퍼를 통해서 슈퍼클래스의 메서드를 호출또는 참조하는 표현식만 허용됨으로, 항상 메서드 이름이 이어진다.
this 와, super 를 스택에 올리는 바이트코드를 에밋하고 이어서 OP_GET_SUPER 와 메서드 이름을 에밋한다.
OP_GET_SUPER 를 인터프리팅 할때 수퍼 클래스와 메서드가 바인드 된다.

기존 메소드 호출과 마찬가지로 바이드메서드를 힙에 올린후에 호출하는 오버헤드를 없애기 위해서 전용 옵코드인 OP_SUPER_INVOKE 를 만든다.
슈퍼클래스의 메서드를 호출하는 표현식은 이 옵코드를 에밋하도록 컴파일 된다.
기존 메서드 호출의 스택 컨벤션에 일치시키기 위해서 메서드의 인수가 먼저 스택에 올라가고 그 다음 슈퍼 클래스의 이름이 스택에 올라가도록 바이트코드가 에밋된다.
OP_SUPER_INVOKE 의 인터프리팅에서는 수퍼클래스를 스택에서 읽어와서 invokeFromClass 로 바로 실행시킨다. 이때 인수는 스택에 이미 준비되어 있다.
